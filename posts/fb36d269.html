hexo source backup

发表于 2021-02-20 更新于 2021-06-21 分类于 hexo 阅读次数：

一、前言

为了在不同的设备写博客，以及备份博客源代码，利用github分支来备份hexo。

机制

由于 hexo d部署到github的是hexo编译后的文件, 编译后的文件本地是在.deploy_git/，是用来生成网页的而不是hexo的源文件
所以利用git分支将源文件上传到其他分支备份即可

二、备份

  1. 首先在原来网页的仓库中创建一个新分支
    backup

  2. 将backup设为默认分支

  3. 新建一个blog文件夹把新分支仓库克隆到本地
    git clone ...

  4. 将 blog/下除了 .git文件夹外的其他文件删除

  5. 把hexo源文件除了 .deploy_git 都复制到新文件夹 blog/ 中

    要注意在theme中是否有嵌套 .git 文件夹, 有就删去，git不允许嵌套上传

  6. 将源文件上传至 backup 分支

三、恢复

  1. 安装git
    sudo pacman -S git

  2. 设置git邮箱和用户名

    git config –global user.name “your githubname”
    git config –global user.email “your email”

  3. 设置ssh key

    ssh-key -t rsa -C “your email”
    验证
    ssh -T git@github

  4. 安装nodejs && npm && npm-check && npm-upgrade
    sudo pacman -S nodejs npm
    用来检测升级模块
    npm i -g npm-check
    npm i -g npm-upgrade

  5. 安装hexo
    sudo npm i -g hexo-cli
    在需要的文件夹中把 backup 仓库克隆到本地
    进入文件夹执行
    npm i
    npm i hexo-deployer-git git工具

  6. 生成部署
    hexo clean
    hexo g
    hexo d

  7. 备份
    git add .
    git commit -m "backup"
    git push

  8. 远端同步
    git pull
